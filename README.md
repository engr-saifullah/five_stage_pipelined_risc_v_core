# 32-bit 5-Stage Pipelined RISC-V Processor (RV32I)

A **32-bit 5-stage Pipelined RISC-V (RV32I) Processor** designed and implemented in **SystemVerilog**, featuring **data hazard resolution through forwarding**, **control hazard handling using stall and flush mechanisms**, and functional verification using **Xilinx Vivado**.

This project demonstrates the implementation of a complete pipelined processor architecture and serves as a practical introduction to modern CPU microarchitecture and RTL design.

---

## Project Overview

This processor implements the standard **five-stage RISC-V pipeline**, enabling multiple instructions to execute concurrently while maintaining correctness through hazard detection and resolution.

The project emphasizes modular RTL design, efficient pipeline control, and verification using simulation.

---

## Pipeline Stages

The processor consists of the following pipeline stages:

| Stage | Description |
|-------|-------------|
| IF | Instruction Fetch |
| ID | Instruction Decode & Register Read |
| EX | Execute / ALU Operations |
| MEM | Data Memory Access |
| WB | Register Write Back |

# Hazard Handling

## Data Hazards

The processor resolves RAW (Read After Write) hazards using a dedicated **Forwarding Unit**.

Features:

- EX → EX Forwarding
- MEM → EX Forwarding
- Automatic operand selection
- No unnecessary pipeline stalls

Example:

```assembly
add x1, x2, x3
sub x4, x1, x5
```

The result from the ADD instruction is forwarded directly to the SUB instruction without waiting for write-back.

---

## Load-Use Hazard

Forwarding alone cannot resolve load-use dependencies.

Example:

```assembly
lw x1, 0(x2)
add x3, x1, x4
```

The Hazard Detection Unit:

- Detects dependency
- Stalls the pipeline
- Inserts one bubble (NOP)
- Resumes execution safely

---

## Control Hazards

Control hazards generated by branches and jumps are resolved using:

- Pipeline Stall
- Pipeline Flush

When a branch is taken:

- Incorrect instructions already fetched are flushed.
- The Program Counter is redirected to the correct target address.
